2. SPECIFIC BUG RESOLUTIONS & ARCHITECTURAL UPGRADES

[A] PREVENTING SPEECH CAPTURE SELF-TALK FEEDBACK (THE NEURAL LINK FIX)
* Problem: The previous implementation unmuted the microphone as soon as the
  first binary audio packet was received from the server. TTSService's playAudio()
  is synchronous and returns immediately while the sound card continues to
  play the actual audio. Consequently, J.A.R.V.I.S would hear, transcribe, and
  reply to his own voice, triggering an endless self-talk loop.
* Solution: Implemented a background reactive monitor in `useBrain.js`. It
  checks the hardware playback state `TTSService.isPlaying()` and the local
  buffer `TTSService.audioQueue.length` every 100ms. It keeps the speech recognition
  engine strictly silenced until all audio queue buffers are fully drained.

[B] DYNAMIC WEBSOCKET URL RESOLUTION (THE WS LINK FIX)
* Problem: Hardcoded `localhost:8000` URLs restricted the developer from
  running or testing J.A.R.V.I.S on alternate ports, local IP addresses (LANs
  for mobile testing), or custom domains.
* Solution: Replaced all static references with a dynamic URL resolver mapping
  directly to the window context hostname while preserving environment variables:
  ```javascript
  const protocol = window.location.protocol;
  const hostname = window.location.hostname || 'localhost';
  const API_URL = `${protocol}//${hostname}:8000`;
  ```
* Reconnection System: Built exponential backoff with jitter (initial delay 500ms,
  max 10s, 1.5x decay, ±30% randomized jitter) to protect the server from 
  thundering herd reconnections. Added tab-visibility and online event handlers 
  to trigger immediate connection restoration.

[C] INSECURE CONTEXT & USERMEDIA GUARDS (SECURE DEVICE SAFEGUARDS)
* Problem: Modern browsers (Chrome, Firefox, Safari) strictly limit microphone
  access (`navigator.mediaDevices.getUserMedia`) to secure contexts (HTTPS or 
  localhost). Accessing the application on any other HTTP address resulted in
  unhandled promise rejections, crashing the HUD startup sequence.
* Solution: Injected validation checks during initial state rendering and stream
  initiation. The system now alerts the console about insecure context issues
  and rejects the initialization promise cleanly with a descriptive, user-friendly
  warning rather than crashing the React DOM rendering.
* Files Modified:
  - e:\Download\Omkar\J.A.R.V.I.S\frontend\src\hooks\useSpeech.js

[D] MEMORY CONCATENATION DEFENSIVE BOUNDARIES
* Problem: The LLM prompt streaming function (`stream_llm` in `processor.py`)
  appends `relevant_context` directly to the system prompt string. If the memory
  retrieval function returned `None` or an unexpected type, the string 
  concatenation would throw a fatal `TypeError` and sever the WebSocket connection.
* Solution: Secured the concatenation expression to default safely to an empty
  string when memory query responses are nullish:
  ```python
  messages = [{"role": "system", "content": self.system_prompt + (relevant_context or "")}]
  ```
