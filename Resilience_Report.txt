1. EXECUTIVE SUMMARY
================================================================================
v3.2 represents a hardening + cleanup pass on top of the v3.1 resilience
baseline. Three engineering tracks completed:

  TRACK A — EXECUTION ISOLATION
    The terminal-execution tool no longer touches the host OS environment
    directly. All shell commands are routed through `core.sandbox.run_sandboxed`,
    which selects between three backends at runtime:
      * Docker (`--network=none --read-only --cpus=0.5 --memory=256m
        --pids-limit=64`)
      * Windows Job Object (`KILL_ON_JOB_CLOSE` + CPU/memory/active-process
        caps via pywin32)
      * Hardened subprocess (stripped env, locked CWD, restricted PATH,
        hard timeout)

  TRACK B — WIRE-PROTOCOL REFACTOR
    Control parameters are no longer embedded as `[TOOL_START:…]` sentinels
    inside the streaming token text. Three new typed JSON frames carry the
    same information out-of-band:
      {"type":"tool_lifecycle", "state":"start"|"end", "name":"<UPPER>"}
      {"type":"memory",         "state":"active"}
      {"type":"vision",         "state":"start"|"end"}

  TRACK C — PROJECT FOOTPRINT LOCKDOWN
    * Recreated the broken uv `.venv` trampoline.
    * Pinned dependencies: `backend/requirements.lock.txt` (110→113 entries).
    * Wired the previously-orphaned `NexusSecureStorage` class to a FastAPI
      router (`backend/nexus_routes.py`) — the React `NexusVault` panel can
      now actually reach the backend.
    * Deleted 13 useless artefacts from the working tree (see §16).
    * `cryptography` added to requirements (was a hidden runtime dep).

2. ORIGINAL RESILIENCE FIXES (v3.1)
================================================================================
[A] SELF-TALK FEEDBACK ELIMINATION (THE NEURAL LINK FIX)
    Problem  : Mic re-armed the instant first MP3 frame arrived; TTSService
               .playAudio() returned immediately while the sound card kept
               playing — J.A.R.V.I.S transcribed his own voice into an
               infinite loop.
    Solution : Reactive 100 ms polling monitor in useBrain.js that gates
               speech recognition on
                   TTSService.isPlaying() === false
                   AND
                   TTSService.audioQueue.length === 0
    Files    : frontend\src\hooks\useBrain.js

[B] DYNAMIC WEBSOCKET URL RESOLUTION
    Problem  : Hardcoded `ws://localhost:8000` blocked LAN / mobile / custom
               host testing.
    Solution : Dynamic resolver — honours REACT_APP_API_URL override, else
               derives from window.location:
                   `${protocol}//${hostname}:8000`
               Reconnect: exponential backoff (500 ms → 10 s, ×1.5 decay,
               ±30 % jitter) + tab-visibility & online-event triggers +
               15 s ping / 5 s pong heartbeat.
    Files    : frontend\src\hooks\useBrain.js

[C] INSECURE-CONTEXT / getUserMedia GUARDS
    Problem  : `navigator.mediaDevices.getUserMedia` rejects outside HTTPS
               or localhost — unhandled promise rejection crashed the HUD.
    Solution : Pre-flight `window.isSecureContext` check + explicit
               capability guard; failure now throws a friendly Error
               instead of corrupting React render state.
    Files    : frontend\src\hooks\useSpeech.js

[D] MEMORY CONCATENATION DEFENSIVE BOUNDARIES
    Problem  : `system_prompt + relevant_context` raised TypeError when
               memory recall returned None, severing the WS connection.
    Solution : Null-coalesce the recall payload (post-conflict-resolution
               final form):
                   content = self.system_prompt + (("\n" + recall) if recall else "")
    Files    : backend\core\processor.py

3. v3.2 HARDENING (E-H)
================================================================================
[E] EXECUTION ENVIRONMENT ISOLATION
    Problem  : `_h_execute_terminal` invoked `subprocess.run` directly with
               only an allow-list + metachar filter. A buggy or compromised
               allow-listed binary could fork uncontrolled descendants,
               escape the parent's lifetime, or read the host environment
               (including the Groq key).
    Solution : New module `backend/core/sandbox.py`. Single entry point
               `run_sandboxed(command, timeout, max_output)` with three
               backends and runtime auto-selection. Allow-list and
               metacharacter filter re-validated INSIDE the sandbox.
    Files    : backend\core\sandbox.py (new, 320+ LOC)
               backend\core\tool_registry.py (handler rewired)
               backend\core\security.py (run_safe_command removed —
                                         superseded by sandbox)
               backend\requirements.txt (pywin32 added for Windows backend)

[F] WIRE-PROTOCOL REFACTOR — STRUCTURED CONTROL FRAMES
    Problem  : `stream_llm` emitted sentinel strings inside the token
               stream (`[TOOL_START:NAME]`, `[MEMORY_ACTIVE]`,
               `[VISION_ACTIVE]`). The HUD had to parse the text stream
               for control transitions — fragile against user text
               containing `[`, and inseparable from rendering.
    Solution : `stream_llm` now yields typed dicts:
                 {"kind":"token", "data":"..."}
                 {"kind":"event", "type":"tool_lifecycle",
                                 "state":"start"|"end", "name":"<NAME>"}
                 {"kind":"event", "type":"memory", "state":"active"}
                 {"kind":"event", "type":"vision", "state":"start"|"end"}
               The `/ws` dispatcher maps each event 1:1 to a JSON frame.
               The frontend `useBrain.js` switch handles each as its own
               `case` (no string parsing inside `streamingText`).
    Files    : backend\core\processor.py
               backend\main.py
               frontend\src\hooks\useBrain.js

[G] NEXUSVAULT WIRING — Z-KNOWLEDGE STORAGE GOES LIVE
    Problem  : `frontend/src/component/NexusVault.js` calls 6 endpoints
               under `/api/nexus/*`. The `NexusSecureStorage` class
               existed in `backend/core/secure_storage.py` but no FastAPI
               router exposed it. Every action returned 404.
    Solution : New `backend/nexus_routes.py` — APIRouter with the 6
               endpoints, Pydantic body schemas (min password length,
               UUID length cap), `Annotated[..., File()]` / `Form()`
               idioms, RFC-5987 filename headers for downloads,
               appropriate HTTP status mapping (401 wrong password,
               404 missing UUID, 409 already initialised, 413 upload
               too large). Router mounted in `main.py` via
               `app.include_router(nexus_router)`.
    Files    : backend\nexus_routes.py (new)
               backend\main.py
               backend\core\secure_storage.py (logger.warning instead of
                                               print; narrowed except
                                               Exception to (ValueError,
                                               OSError))
               backend\requirements.txt (cryptography added)

[H] PROJECT FOOTPRINT LOCKDOWN
    Problem  : The `backend/.venv/` was a broken uv trampoline ("uv
               trampoline failed to spawn Python child process"). No
               lockfile existed; dependency drift was inevitable. Source
               tree contained Maven-Nexus files (POM, settings.xml with
               cleartext `admin123` password, deploy.ps1) that had nothing
               to do with the AI project — stale leftovers that fooled
               grep for "nexus" into thinking it was the AI vault.
    Solution : Recreated `.venv` via `py -3 -m venv backend\.venv`,
               installed deps, captured exact pins to
               `backend/requirements.lock.txt` (113 entries including
               cryptography + pywin32).
               Deleted: pom.xml, settings.xml, deploy.ps1, path.txt,
               scratch/dump_memory.py, backend/scratch/purge_stark.py,
               backend/scratch/test_security.py (broken — called methods
               that no longer existed), all __pycache__/, .ruff_cache/,
               .sandbox_run/, vault/, frontend/build/.

10. NEXUSVAULT CRYPTOGRAPHIC PROFILE
================================================================================
Algorithm     : AES-256-GCM (16-byte tag, authenticated encryption).
KDF           : PBKDF2-HMAC-SHA256, 100 000 iterations.
Salt          : 16 bytes, `os.urandom`, per-encryption.
IV            : 12 bytes, `os.urandom`, per-encryption.
Layout        : SALT(16) || IV(12) || CIPHERTEXT
Storage       : <repo>/vault/index.enc + <repo>/vault/<uuid>.enc per file.
Tampering     : GCM auth-tag mismatch → ValueError (HTTP 401 at the router).
Zero-knowledge: master password held only in request scope; never persisted.
Secure shred  : on delete, file is overwritten with os.urandom (size-matched),
                f.flush(), os.fsync(fileno()), then unlinked.
                SSD wear-levelling caveat applies — true shred requires
                whole-drive wipe or hardware secure-erase.

14. WEBSOCKET WIRE PROTOCOL (v3.2 structured frames)
================================================================================
Inbound JSON (client → server)
  {"type":"chat",          "text":"..."}            user turn (≤ 4000 chars)
  {"type":"clear_history"}                          wipe rolling window
  {"type":"ping",          "timestamp":<num>}       client heartbeat

Outbound JSON (server → client)
  {"type":"status",        "data":"idle|thinking|speaking|history_cleared"}
  {"type":"token",         "data":"<token>"}        streamed LLM chunk
  {"type":"response_end",  "data":"<full text>"}    final transcript
  {"type":"audio_start"}                            next binary frames = MP3
  {"type":"telemetry",     "data":{"cpu":<f>,"ram":<f>,"status":"nominal|warning|critical"}}
  {"type":"pong",          "timestamp":<num>}
  {"type":"error",         "data":"<message>"}
  {"type":"tool_lifecycle","state":"start"|"end","name":"<UPPER_TOOL_NAME>"}  ★ NEW
  {"type":"memory",        "state":"active"}                                  ★ NEW
  {"type":"vision",        "state":"start"|"end"}                             ★ NEW

Outbound binary
  Raw MP3 frames (Edge TTS output). Serialised via asyncio.Lock so no
  binary frame can interleave with a JSON token frame.

DEPRECATED in v3.2 (no longer emitted):
  In-stream sentinel strings [MEMORY_ACTIVE], [VISION_ACTIVE],
  [VISION_ENDED], [TOOL_START:<NAME>], [TOOL_END:<NAME>]. These are now
  carried as dedicated structured frames above.

17. KNOWN BUGS, OPEN ERRORS, LATENT RISKS
================================================================================
[BUG-01] No server-side timeout on stream_llm.
         Symptom: if Groq stream stalls between server and Groq, the WS
                  coroutine parks indefinitely. The client heartbeat
                  catches a fully-dead connection, but not a slow trickle.
         Fix    : wrap with `asyncio.wait_for(processor.stream_llm(...),
                  timeout=N)`.

[BUG-02] No client-side tool watchdog.
         Symptom: if `tool_lifecycle:start` is delivered but `:end` never
                  is (server crash mid-tool), the HUD stays in tool_use
                  until the heartbeat reconnects.
         Fix    : per-tool 30 s timer in useBrain.js — set on
                  `state==="start"`, clear on `state==="end"`, reset
                  pipelineState + force reconnect on expiry.

[BUG-03] Sandbox windows_jobobject backend races on subprocess spawn.
         Symptom: between Popen and AssignProcessToJobObject (microseconds),
                  the child could fork descendants outside the job.
         Mitigation in place: allow-list excludes every cheap fork-spawner
                  (cmd, powershell, bash, sh, perl, ruby, node REPL).
         Hard fix: rewrite via win32process.CreateProcess with
                  CREATE_SUSPENDED + ResumeThread (Popen doesn't expose
                  thread handle).

[BUG-04] Vosk model dir absence is silent.
         Symptom: `_load_vosk_model` logs INFO and returns None. The
                  cloud STT path then loses its safety net without any
                  HUD signal.
         Fix    : surface availability in /health and emit a startup
                  banner on the WS.

[BUG-05] Telemetry per-session psutil.cpu_percent cost.
         Symptom: every WS session spawns its own 5-second telemetry loop.
                  At N clients, cpu_percent runs N times per 5 s.
         Fix    : centralise telemetry in a single asyncio task; fan-out
                  to subscribers.

[BUG-06] No rate limiting on /ws or REST endpoints. One curl loop can
         drain the Groq token budget.
         Fix    : SlowAPI or starlette-limiter; per-IP and per-session
                  caps.

[BUG-07] redact() doesn't cover all common keys. Currently matches Groq
         `gsk_*`, OpenAI/generic `sk-*`, plus a generic key/token/secret
         regex. Missing: Anthropic `sk-ant-*`, Google `AIza*`, AWS `AKIA*`.
         Fix    : extend `_SECRET_PATTERNS`.

[BUG-08] NexusVault secure-shred is best-effort on SSD.
         Symptom: TRIM + wear-levelling mean the overwritten block may
                  not be on the same physical cell. Cryptographic shred
                  is the safest fallback (lose the master password).
         Fix    : (informational) — document the limitation in UI; user
                  may want full-disk encryption underneath.

[BUG-09] No CSRF protection on NexusVault routes. Defended in practice
         by the same-origin policy + CORS allow-list, but a same-origin
         XSS would be sufficient to invoke the vault endpoints.
         Fix    : add a CSRF token tied to the unlock session.

[BUG-10] processor.stream_llm catches `Exception` broadly. A rate-limit
         error, a token-budget overflow, and a network drop all surface
         as the same fallback line.
         Fix    : typed handling per groq.* exceptions + structured
                  `{"type":"error","code":...}` frame.

[BUG-11] frontend/.eslintrc.js previously had
         `'react-hooks/exhaustive-deps': 'off'`. That hid a real
         missing-dep bug in App.js#handleInitialize (caught & fixed).
         Now set to `'warn'`. Leave it on.

[BUG-12] processor.py mixes "Summarise"/"sir" Indian-English voice with
         a couple of Hinglish-tinged exception strings. Localisation
         strategy is undocumented — treat as cosmetic for now.

[BUG-13] export_conversation schema lacks `"required": ["format"]`.
         Groq may emit empty args, falling through to implicit "pdf".

[BUG-14] backend\temp\ purge runs once at boot. Long-running sessions
         that export many PDFs see the dir grow until next restart.
         Fix    : asyncio task on 10-minute cadence.

19. OPERATIONAL NOTES FOR FUTURE MAINTAINERS
================================================================================
* `npm run sync-env` is auto-invoked by dev/start/backend — never edit
  frontend/.env or backend/.env directly; they're overwritten on every
  boot from root .env.

* Mic-mute-during-TTS pattern in useBrain.js is load-bearing. Do not
  inline-optimise the 100 ms polling away without replacing it with an
  event-based equivalent (TTSService would need a `playbackDrained`
  event).

* The two-pass LLM call (tool_choice=auto then stream) doubles the Groq
  round-trip on every turn. Do not collapse into one pass without
  re-deriving how tool results re-enter the message list.

* core.sandbox is the SINGLE entry point for shell execution. Never call
  subprocess.run from a tool handler. The allow-list is re-validated
  inside the sandbox, but bypassing it skips the env scrub, locked CWD,
  and job-object lifetime control.

* NexusVault master password is *never* persisted server-side. Forgetting
  it is permanent loss. The encrypted blobs without the password are
  computationally infeasible to decrypt — that's the point.

* `react-hooks/exhaustive-deps` is `'warn'` in frontend/.eslintrc.js.
  Leave it on. Previously was 'off' and hid a real missing-dep bug.

* Pong logs (`type:"pong"`) are intentionally silent on the client
  console; do not re-enable without a feature flag.

* The v3.2 structured-frame refactor means the HUD must never depend on
  the literal sentinel strings `[TOOL_START:...]` etc. in `streamingText`.
  They're gone.
